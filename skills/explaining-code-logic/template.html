How Binary Search Works

Binary search finds a target in a sorted list by repeatedly halving the part of the list it still needs to look at.

Why it exists: checking every item one by one is slow for large lists. Binary search throws away half the remaining items on every step, so a list of a million only takes ~20 checks instead of a million.

Like finding a word in a dictionary

You don't read page 1, 2, 3… You flip to the middle. Too far? Ignore the whole second half. Not far enough? Ignore the first half. Each flip discards half the book — exactly what binary search does to the list.

The loop at a glance

① Look at the middle
→
② Found it?
→
③ Too small? go right
→
④ Too big? go left, repeat

Track two ends — a low and a high pointer. Look at the item halfway between them.

Think of it as opening the book right in the center of the pages you haven't ruled out yet.

mid = (low + high) // 2
guess = arr[mid]

If the middle item equals what you're looking for, you're done — return its position.

if guess == target:
    return mid   # found it

If the middle item is smaller than the target, the target can only be to the right. Move low just past the middle.

elif guess < target:
    low = mid + 1

If the middle item is bigger, the target is to the left, so move high just before the middle. Then loop back to step 1 with the smaller range. Stop when low passes high — the target isn't there.

else:
    high = mid - 1
# loop continues while low <= high

Check yourself

The list is [2, 5, 8, 12, 16, 23, 38] and you're searching for 5. The middle item is 12. What happens next?

12 is bigger than 5, so we throw away everything from 12 rightward (step 4) and search only [2, 5, 8]. Next middle is 5 → found on the very next check.
